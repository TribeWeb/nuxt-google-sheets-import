## Module options

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['nuxt-google-sheets-import'],
  googleSheetsImport: {
    apiBase: '/api/google-sheets-import',
    googleApiKeyRuntimeKey: 'googleApiKey',
    defaultContentDir: 'content/data'
  }
})
```

- `apiBase`: base route for module API handlers
- `googleApiKeyRuntimeKey`: runtime config key used to resolve your Google API key
- `defaultContentDir`: default write target for imported files

## Built-in route

The module registers a page at:

```text
/google-sheets-import
```

This page provides three tabs:

- `Setup Google Sheet`
- `Import data`
- `Export data`

## Schema source contract

Define this file in the consuming app:

```ts [utils/googleSheetImportSchemas.ts]
export const schemas = {
  machines,
  materials
}
```

At runtime, this is auto-imported as `googleSheetsImportSchemas`.

## API endpoints

Base path: `{apiBase}` (default `/api/google-sheets-import`)

- `GET /sheets`
  - Query: `spreadsheetId`
  - Returns available sheet tabs.
- `POST /values`
  - Validates and transforms sheet values against a selected schema.
- `GET /collection-type`
  - Resolves whether a schema maps to a `page` or `data` collection.
- `GET /schema-columns`
  - Query: `schema?`
  - Returns:
    - `schemas`
    - `columns`
    - `collectionType`
    - `pageOverrideColumns`
- `POST /write`
  - Writes transformed records to markdown frontmatter, JSON, or YAML.

## Export behavior details

- Clipboard copy in export uses TSV for reliable Google Sheets paste.
- File download uses quoted CSV for file-based data exchange.

## Write behavior

Supported overwrite strategies:

- `overwrite`
- `skip`
- `overwrite-frontmatter`
