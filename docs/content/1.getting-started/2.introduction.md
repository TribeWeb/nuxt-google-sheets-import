Welcome to **Google Sheets Import**, a Nuxt module for seamless integration between your Nuxt Content frontmatter and Google Sheets.

## Why Google Sheets Import?

There are several ways to manage and edit frontmatter in a Nuxt Content project, but user focussed options such as those offered by Nuxt Studio involve individual manual file edits, which can be time-consuming and error-prone.

Whether you're a developer or client, initial setting up of multiple pages or data records can be particularly tedious, especially when creating catalogs, inventories, or other structured content.

In-app bulk editing is relatively straightforward but it requires manipulation of an IDE by a developer or writing custom code which must be maintained.

Google Sheets Import leverages the familiarity and collaborative features of Google Sheets to manage frontmatter data.

This greatly simplifies the bulk editing process and also allows for collaboration with team members without developer experience but who have ownership and knowledge of the data being used.

For example, developers, product managers and clients can work together to best understand how each attribute of a product should be defined, constrained and reused for maximum effect.

## How it works

Nuxt Content already requires that each page type or data set is defined using Zod schemas. These provide a clear structure for content frontmatter.

Google Sheets Import builds on this by showing you how to map your Zod schemas to columns in a Google Sheet. Content can be created, adapted and manipulated by in a spreadsheet and then imported as `.md`, `.json` or `.yml` files.

When existing files require bulk editing of frontmatter data, they can be exported. After being pasted or uploaded into Google Sheets, the spreadsheets can then be reimported. All without losing any markdown in `.md` files.
