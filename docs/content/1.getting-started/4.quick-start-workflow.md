After installing the module, use this workflow to go from schema definition to imported files.

## 1. Define schema source

Create `~/utils/googleSheetImportSchemas.ts`:

```ts [utils/googleSheetImportSchemas.ts]
export const schemas = {
  machines,
  materials
}
```

The module auto-imports this as `googleSheetsImportSchemas` in app and server runtime.

## 2. Ensure matching Nuxt Content collections

Each schema key should map to an existing Nuxt Content collection in `content.config.ts`.
Collection type (`page` vs `data`) is resolved from your collection definitions.

## 3. Open the module page

Start your app and open:

```text
/google-sheets-import
```

This page uses tabs for the full workflow:

- `Setup Google Sheet`
- `Import data`
- `Export data`

## 4. Setup Google Sheet headers

In `Setup Google Sheet`:

- Select your schema.
- Copy expected column names.
- Optionally copy page override columns for `page` collections.

## 5. Import rows into content files

In `Import data`:

- Choose spreadsheet and sheet range.
- Validate rows against your selected schema.
- Write to frontmatter, JSON, or YAML with your desired overwrite strategy.

Supported overwrite strategies:

- `overwrite`
- `skip`
- `overwrite-frontmatter`

## 6. Export existing content back to sheets

In `Export data`:

- Load records from a collection.
- Copy rows as TSV for direct Google Sheets paste.
- Or download quoted CSV for file-based workflows.

## Suggested cell value patterns

- `string`: `example text`
- `number`: `123`
- `boolean`: `true` / `false`
- `enum` / `literal`: one of the allowed values
- `date-like string`: `2026-01-01`
- `string[]`: `foo, bar, baz` in one cell
- `object[]`: indexed headers such as `items[0].name`, `items[0].price`

## Quick endpoint check

To confirm API key + sheet access are working:

```bash
GET /api/google-sheets-import/sheets?spreadsheetId=<SPREADSHEET_ID>
```
