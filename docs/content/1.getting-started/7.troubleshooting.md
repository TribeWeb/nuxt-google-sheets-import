## Google API errors

### `403 PERMISSION_DENIED` / `The caller does not have permission`

- Cause: sheet is not publicly readable.
- Fix: `Share` -> `General access` -> `Anyone with the link` + `Viewer`.

### `403 API key not valid` / `API has not been used in project`

- Cause: key is wrong, API not enabled, or key restrictions are incorrect.
- Fix: enable Google Sheets API and verify API restriction includes it.

### `403 Requests from this referrer/IP are blocked`

- Cause: key application restrictions do not match execution environment.
- Fix: update restrictions (`IP addresses` recommended for server-side use).

### `404 Requested entity was not found`

- Cause: malformed or incorrect spreadsheet ID.
- Fix: copy ID from `https://docs.google.com/spreadsheets/d/<SPREADSHEET_ID>/edit`.

### `400 Unable to parse range`

- Cause: invalid A1 range or tab name typo.
- Fix: verify tab name and use ranges like `A:Z`.

## Module workflow issues

### Schema is missing in UI or API

- Ensure `~/utils/googleSheetImportSchemas.ts` exists.
- Ensure it exports `schemas`.
- Restart dev server after schema/source changes.

### Import writes to unexpected collection type

- Collection type is derived from Nuxt Content collection definitions.
- Verify your collection key and schema key mapping in `content.config.ts`.

### CSV copy pastes poorly into Google Sheets

- Use module's `Copy for Google Sheets` action in Export tab.
- It copies TSV specifically for horizontal Sheets paste.

## Local development checks

From repository root:

```bash
npm install
npm run dev:prepare
npm run test
```

Run playground app:

```bash
npm run dev
```

Run docs site:

```bash
cd docs
npm install
npm run dev
```

## Release checklist

- Run `lint`, tests, and build checks.
- Verify import and write flows in playground.
- Verify docs pages reflect current behavior.

Release command:

```bash
npm run release
```
