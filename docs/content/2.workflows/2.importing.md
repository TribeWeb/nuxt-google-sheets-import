Use this workflow to validate and write Google Sheet rows into Nuxt Content files.

Before importing:

- Ensure schemas are defined in `app/utils/googleSheetsImportSchemas.ts`.
- Ensure the same schema keys are used in `content.config.ts` collections.
- Ensure your Google Sheet header row maps to schema fields.

::steps
### Go to the module page

With your application running in `dev`, navigate to `/google-sheets-import` and select the `Import data` tab.

### Pick spreadsheet source details

Choose:

- Spreadsheet ID
- Sheet tab/title
- A1 range (for example `A:Z`)
- Schema key

The module checks these values before attempting to write files.

### Select a schema

Select the schema key that should validate each row.

  :::note
  If you do not see your schema listed, ensure it is [configured correctly](/getting-started/schema-setup).
  :::

### Validate rows

Run the import step to fetch values and validate against the selected schema.

Rows that fail validation are reported with details so you can fix source cells before writing output.

### Assign filename stems & ordering prefix

- Choose the column to use as filename stem (without extension)
- Optionally, choose the column to use as ordering prefix

### Choose output format and write strategy

Select your output mode:

- Frontmatter (`.md`)
- JSON (`.json`)
- YAML (`.yml`)

Then select overwrite behavior:

- `Skip existing files`
- `Overwrite frontmatter only (.md files)`
- `Overwrite all`

Write files into your selected content folder.

### Verify generated content

After writing, verify the generated files under your content directory and confirm they are loaded by Nuxt Content.
::

## Header and value guidance

- Use exact schema property names in header cells.
- Nested arrays/objects can use indexed keys like `items[0].name`.
- For array-like string fields, use delimited values per your schema parsing strategy.
- For booleans, prefer explicit values like `true` or `false`.

:::note
See the [API reference](/runtime-api-reference#suggested-cell-value-patterns) for detailed header and value formatting guidance to ensure successful imports.
:::

## Overwrite behavior definitions

- `Skip existing files` (`skip`): if a target file already exists, it is left unchanged.
- `Overwrite frontmatter only (.md files)` (`overwrite-frontmatter`): for Markdown output, the frontmatter is replaced and the existing markdown body is kept.
- `Overwrite all` (`overwrite`): if a target file exists, the full file content is replaced.

## API equivalent

The UI workflow is built on these routes:

- `GET /api/google-sheets-import/sheets`
- `POST /api/google-sheets-import/values`
- `POST /api/google-sheets-import/write`

## Page collection overrides

When importing into `page` collections, you can include [schema overrides](https://content.nuxt.com/docs/collections/types#schema-overrides) such as `title` or `seo` directly as columns.

These fields do not need to be defined in your Zod schema.
