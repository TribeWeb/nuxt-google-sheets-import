Use this workflow to export existing Nuxt Content records into spreadsheet-friendly formats.

The export tool flattens nested objects into key paths so complex records can still be moved into tabular form.

::steps
### Go to the module page

With your application running in `dev`, navigate to `/google-sheets-import` and select `Export data`.

### Select a schema

Choose the schema/collection you want to export.

  :::note
  If you do not see your schema listed, ensure it is [configured correctly](/getting-started/schema-setup).
  :::

### Load records

Click `Load records` to fetch all rows from the selected collection.

The module generates a unified header list from all row keys and flattens values to strings for export.

### Choose export output

You can:

- Copy TSV (recommended for direct Google Sheets paste)
- Download CSV (quoted file download)

### Paste or import into Google Sheets

For quick use, paste TSV directly into a blank sheet.

For file workflows, import the downloaded CSV into Google Sheets.

### Review and iterate

If you update records in content, reload and export again to keep spreadsheet data in sync.
::

## What gets exported

- Rows come from the selected Nuxt Content collection.
- Headers are the union of all flattened keys found across records.
- Missing values for a row/header are exported as empty strings.

## Practical notes

- TSV is usually better for clipboard workflows.
- CSV is better for versioned file hand-off.
- Large collections may take longer to load; wait for `Export prepared` status.

## API equivalent

The export view uses these capabilities:

- `GET /api/google-sheets-import/schema-columns` to populate schema options
- Nuxt Content collection querying to read record data

::note
To prepare [Nuxt Content page schema overrides](https://content.nuxt.com/docs/collections/types#schema-overrides) such as `title` or `seo` for round-trips, include those columns in the sheet where relevant.
::
